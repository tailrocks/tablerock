## Findings considered and rejected

- **`PathLease` is process-local, not a cross-process file lock**
  (`persistence/src/lib.rs:339-366`): real, but single-owner-per-process is
  the documented Phase 2 scope; cross-process guarding is recorded open in
  evidence (`persistence/60` "cross-process application ownership remains
  open") and lands with Phase 11 hardening (plan 018 failure-injection).
  Not a standalone plan.
- **`support_facts` table dormant** (migration 0002, no production
  reader/writer): harmless; Phase 15 support-diagnostics work (plan 021)
  will either use or retire it. Not worth a plan.
- **ClickHouse `connect` does no network round-trip** (`clickhouse.rs:216`):
  by design of the official client; Test/health round-trips cover the UX
  need (plans 002/006). Not a bug.
- **Sync/blocking persistence actor + synchronous CH cancellation budget**:
  deliberate, evidence-recorded tradeoffs (docs 58/129/135). Do not
  "modernize".
- **Read-path secret decode intermediates not individually zeroized**
  (`profile_store.rs:825` builds core types from transient `String`/`Vec`):
  bytes move into zeroizing wrappers immediately; residual copies are a
  known Rust-level limitation. Fold into plan 018's secret/log audit rather
  than a dedicated plan.
- **No clap/arg parsing in the CLI**: intentional for a TUI-first binary;
  revisit only if headless subcommands become a requirement (none in the
  parity ledger).
