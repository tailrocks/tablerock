# Phase 2 Object-Safe Driver Adapter

## Decision

The engine now exposes one dyn-compatible `DriverSession` and
`DriverPageStream` boundary across PostgreSQL, ClickHouse, and Redis. Methods
return `Pin<Box<dyn Future + Send + 'a>>`; no `async-trait` macro, executor
dependency, generated function, or concrete client type enters the contract.

This shape follows the Rust Reference: dyn-compatible methods may carry lifetime
parameters and use `&self`, `&mut self`, or `Box<Self>`, but cannot return the
opaque future generated by native `async fn`. Consuming `Box<Self>` gives
shutdown deterministic ownership while borrowed boxed futures support stream
start and page reads.

## Bounded request and page ownership

`DriverPageRequest` is an engine-tagged feasibility enum. Every variant carries
finite `PageLimits` and a per-cell byte cap; Redis additionally bounds scan
count/rounds, and ClickHouse owns a bounded query ID. Custom diagnostics report
the query-ID byte count, never its value.

`DriverPageStream` yields only immutable `ResultPage` values plus completion.
PostgreSQL rows/connection tasks, ClickHouse byte cursors, Redis multiplexed
connections, and all library errors remain private to adapters.

## Failure, cancellation, and shutdown

Cross-engine requests fail with `EngineMismatch` before I/O. Adapter failures
collapse into engine plus a safe class: invalid request, query, connection,
protocol, decode, page, or cancellation transport. Raw errors, SQL, Redis
arguments, query IDs, credentials, and cell values never enter display/debug.

Cancellation accepts an opaque operation ID but currently reports
`Unsupported` for this generic seam because the feasibility sessions do not yet
own operation-to-server-request mappings. Existing PostgreSQL cancellation
evidence remains concrete and server-confirmed. The adapter must not claim
`RequestSent` until task ownership supplies that mapping. Shutdown consumes the
session; PostgreSQL awaits its driven connection task while ClickHouse/Redis
drop their owned clients.

## Evidence

- `adapter_contract.rs` proves engine tagging and redacted request/error output.
- PostgreSQL Testcontainers exercises dyn dispatch, cross-engine rejection,
  explicit unsupported generic cancellation, bounded page streaming, and
  consuming shutdown.
- ClickHouse and Redis Testcontainers start their existing bounded real streams
  through the same dyn-compatible request/page interface.
